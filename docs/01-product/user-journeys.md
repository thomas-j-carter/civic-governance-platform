# User Journeys

Defines key flows.

## 1. Membership Application

1. user visits /membership/application
2. submits application
3. application enters review
4. reviewer evaluates
5. decision made
6. user notified

## 2. Proposal Lifecycle

1. member creates proposal
2. proposal is drafted
3. proposal submitted
4. enters review stage
5. moves to ballot
6. ballot closes
7. certification issued
8. result published

## 3. Record Publication

1. record created
2. record version updated
3. record marked publishable
4. record published
5. visible publicly

## Journey rule

Every journey must:
- map to domain state machine
- map to specs
- be testable
