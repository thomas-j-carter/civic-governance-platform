# Personas

Defines representative user types.

## 1. Public Visitor

- unauthenticated
- consumes public content
- views records and publications

## 2. Applicant

- submits membership application
- tracks status

## 3. Member

- participates in governance
- views internal content
- manages profile

## 4. Reviewer / Moderator

- reviews applications
- manages proposals in early stages

## 5. Governance Operator

- manages proposal lifecycle
- oversees ballots
- executes transitions

## 6. Office Holder

- has formal authority
- participates in governance decisions

## 7. Records / Publication Editor

- manages official outputs
- controls publication state

## 8. Admin

- system-level management
- access control oversight

## Persona rule

Personas are NOT roles.
They are usage archetypes.

Roles are defined in domain layer.
