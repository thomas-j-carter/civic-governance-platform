# Permissions (User Perspective)

Defines what users can do.

## Member

- create proposals
- view internal records
- participate in ballots

## Reviewer

- review applications
- move proposals (limited)

## Governance Operator

- execute lifecycle transitions
- manage ballots

## Admin

- assign roles
- manage system

## Rule

Permissions must:
- align with domain authority model
- not be UI-only constructs
